# 用小白视角测试3种AI编程方案：哪个最适合你的酷炫3D图谱？

---

## 开场：一个视觉魔法的诞生

最近微信刷屏的"手势控制3D图谱"火了。

你是否也被那流畅的三维旋转、手指捏合缩放的视效吸引？我也是。

上一篇文章《5分钟实现手势控制卡片》后，我决定升级这个项目——用同一份设计文档，测试三个不同的AI编程工具，看它们如何把同样的需求变成酷炫的网页应用。

**结果出乎意料**：三个工具走出了完全不同的风格。这次分享不仅是评测，更是一个小白如何与AI协作、迭代升级的故事。关注公众号，后台留言 “hlm”，获得此次代码和测试提示词。

## 测试背景：为什么要用同一份设计文档？

### 测试对象
**红楼梦人物关系3D图谱** - 包含27个人物、40条关系、支持手势控制和搜索功能

这是个复杂度适中的项目：
- ✅ 3D可视化（需要Three.js）
- ✅ 力导向图布局算法（需要物理计算）
- ✅ 手势识别（需要MediaPipe）
- ✅ 交互设计（需要UI/UX品味）

### 测试维度
| 维度 | 说明 |
|------|------|
| **UI/UX体验** | 视觉效果、设计质感、交互流畅度 |
| **调试效率** | 迭代次数、反馈速度、理解成本 |
| **开发成本** | 使用费用、学习曲线 |
| **最终质量** | 功能完整性、性能稳定性 |

### 测试者身份
👤 **前端小白** - 不懂Three.js、不懂WebGL、纯粹从需求出发，看AI如何帮助我完成

这是关键：很多评测都是专业开发者的视角，但小白的需求不同——我们需要**"好看、好用、费用低**

## 🥇 第一名：AMP + Claude Skills Front-end（最具科技感）

### 表现亮点

**UI/UX细节最突出。** 

打开网页的一瞬间，你能感受到设计的用心：
- 深紫色赛博朋克背景 + 星空粒子动画
- 节点发光效果（emissive材质）
- 毛玻璃风格的UI面板（backdrop-filter）
- 文本始终面向摄像机（Billboard效果）
- 三层点光源提供的立体感

这不是"能用"，而是"有品味"。

### 调试体验：Pipeline插件有多丝滑？

这是我最惊喜的地方。

AMP的Pipeline机制让我体验到什么叫"人机协作"：

1. **第一版**: 我给了设计文档，AMP直接用Frontend-Design技能生成完整的HTML+CSS+JS
2. **有问题?**: 我提出"3D球体标签对齐"、"力导向稳定性"等问题
3. **自动优化**: AMP不仅修复问题，还**主动建议改进提示词**，生成了v3.1版本的设计文档
4. **代码质量**: 使用MCP工具组织文件、创建snapshot、自动生成测试报告

感觉就像有个资深开发者在身边，实时听你吐槽，然后给出更好的方案。

**关键数据：**
- 迭代次数：3-4次达到预期
- 反馈延迟：毫秒级
- 代码行数：890行script.js + 240行gesture.js
- 项目大小：仅25KB（高度优化）

### 学到的东西

最宝贵的不是最终代码，而是过程中学到的工程思想：

> "完成后，一定要沉淀一个新的Plan版本。对比初始设计和最终实现，你能看到改进的方向。" 

这就是PDCA循环——Plan(计划) → Do(执行) → Check(检查) → Act(改进)。

AMP帮我把这个循环跑通了，最后生成的COMPLETION_REPORT.md不仅记录了功能实现，还总结了：
- 已验证的功能清单
- 性能优化细节
- 已知限制与改进空间
- 技术栈总结

### 适合人群

✅ **想要高质量效果** + 愿意花时间学习  
✅ **追求设计品味** + 不怕费用  
✅ **需要长期支持** + 想和AI一起成长  

### 成本

有一定费用投入，但如果你用于商业项目、客户展示，这投入立马值回来。

## 🥈 第二名：Gemini App Builder + Flash Preview（最贴心的意外）

### 表现亮点

**我最初对Gemini的期待不高——毕竟UI美化向来不是AI代码生成的强项。**

结果：3-4次迭代后，UI效果也变得不错。

但最让我惊喜的，不是UI本身，而是**它自动融入的AI能力**。

### 意外收获：内容自动增强

传统做法：我们创建3D图谱，点击节点显示人物介绍。

Gemini的想法：为什么不自动为每个人物生成"AI编译的介绍"和"精选原文摘录"？

结果：
- 💡 点击任意人物节点
- 📝 自动调用Gemini生成该人物的简介（融合小说背景 + 现代语言）
- 📖 自动提取小说原文中的经典段落

**用户角度**：这不仅展示了人物，更展示了AI如何理解文学。很贴心。

### 调试过程：需要更多耐心

第一版出来，3D小球变成了平面圆形 + 多层光晕。很……平面。

我提出改进建议 → Gemini修改 → 再看 → 还是不够好 → 再改……

修改了3-4次才达到预期。

不过这个过程也有好处：**每次反馈，Gemini都记住了我的美学偏好**。到最后，代码风格、UI调色都越来越符合"紫色赛博朋克"的气质。

### 为什么这个工具值得关注？

1. **智能化程度高** - 能主动思考"用户可能想要什么"
2. **文化融合** - 对中文文学的理解还不错（虽然有时会过度创意）
3. **成本适中** - Flash Preview模型成本比GPT-4便宜

### 适合人群

✅ **想要省事** + 希望系统更聪明  
✅ **项目需要内容自动增强** + 不怕迭代  
✅ **追求AI赋能** + 而不仅仅是代码生成  

### 成本

作为业内的赛博大善人，哈吉米的Flash-3模型的成本- 免费且质量高， 小白入手应用首选。

## 🥉 第三名：OpenCode（低调稳健的选择）

### 表现亮点

我用"大叔"来形容OpenCode不是贬低，而是尊敬。

**它就像一个靠谱的老工程师**——不花哨，不出幺蛾子，一步步把活儿干好。

### 表现

**中规中矩，但稳如磐石。**

- ✅ 3D图谱渲染：完整
- ✅ 手势识别：可用
- ✅ 动画效果：流畅
- ✅ 性能：稳定

3次调优后，各项功能都能用。没有意外，没有惊喜，但也没有遗憾。

外观是这样的：朴实、稳健、有一种"亲切感"（我用好听的词来形容哈哈）。

### 特色：完全免费

这是最大的卖点——我用了**免费模型**，没有订阅费、没有API计费。

同样的提示词，OpenCode照样实现了：
- 27个人物节点
- 40条关系边
- 手势控制（单手旋转、双手缩放）
- 搜索功能
- UI交互

这对初学者、个人开发者、学生党是个福音。

### 调试体验

流程很顺畅，但没有什么"惊喜"的反馈。

OpenCode的迭代方式是：
1. 我提出需求
2. 它生成代码
3. 代码基本能用
4. 我测试发现问题
5. 它修复问题
6. 重复3-5次，功能完整

没有主动优化建议，没有"为你想到的"细节，但也少了折腾的烦恼。

### 小白友好度

**特别高。**

因为OpenCode的反馈和代码都很直白，没有过度设计，也就没有学习成本。你能更清楚地看到"代码做了什么"，而不是被框架、架构、设计模式绕晕。

### 适合人群

✅ **预算有限**（学生、自由职业者）  
✅ **追求稳定性** + 不需要花哨  
✅ **小白入门** + 想理解代码  
✅ **快速原型验证** + 之后再优化  

### 成本

**稳重稳定，完全免费。** 这改变了一切。当然如果对于资深进阶，可以用ZEN，直接匹配更多模型，每个月10刀，也很实惠了。当然如果你要购买glm4.7 coding套餐，价格是10人民币。萝卜青菜，各取所爱。白嫖党最爱公益站，这也是选择❤️

## 🔥 核心洞察：PDCA循环与AI应用者的风格

现在到了文章最重要的部分。

### 问题：为什么有人用AI效率高，有人效率低？

很多人说"AI代码生成不行"，其实问题不在AI，而在**流程**。

我在这次对比中发现了什么？

**答案就在PDCA循环里。**

### PDCA的四步

1. **Plan（计划）** - 写详细的设计文档、提示词
2. **Do（执行）** - 让AI根据计划生成
3. **Check（检查）** - 测试、对比、发现问题
4. **Act（改进）** - 修改提示词、沉淀经验

**关键是第4步：很多人做完3步就停了。**

AMP的好之处在于，它强制你做第4步。完成项目后，它帮我生成了新版本的design.md（v3.1），记录了：

```
已实现的功能清单
性能优化细节
已知限制与改进方向
```

这不仅是文档，更是**下一次迭代的起点**。

### 这就是AI应用者的核心竞争力

> "人和AI协作多次后，也会形成对应的风格。这个风格在应用开发中，几乎就是灵魂。"  
> —— 类似Paul Graham的观点（好的设计思想来自于深度思考和反复实践）

你用AMP做了10个项目，每个项目都跑完PDCA循环，到第11个项目你的提示词会好10倍。

你的设计品味、工程思想、问题解决能力，这些才是真正的资产。

### 行业大佬怎么说？

Paul Graham在《Maker Schedule Manager Schedule》中提到的"深度思考时间"，现在变成了"与AI深度协作的时间"。

Satya Nadella（微软CEO）说过：**"AI是乘数，不是替代品。关键是人的思想质量。"**

在这次测试中我深刻体会到——

- 🎯 **好的Plan** 决定了AI能帮到什么程度
- 🔄 **频繁的Check** 确保方向不偏
- 📈 **持续的Act** 积累个人风格

这是"AI应用者"和"AI使用者"的根本区别。

## 💡 我的最终建议

**没有绝对的最优选择，取决于你的优先级：**

### 如果你是…

**创业者 / 想要高质量的客户展示**  
→ **选AMP + Claude Skills**  
投入学习成本，完整跑一遍PDCA，你会学到设计思想。

**内容产品经理 / 需要AI内容增强**  
→ **选Gemini App Builder**  
虽然迭代次数多，但最后能给用户惊喜的体验。

**学生 / 个人开发者 / 快速实验者**  
→ **选OpenCode**  
零成本、快速上手、代码清晰易懂。

**我个人的选择：AMP**  
因为我在过程中学到最多，不仅是代码，更是如何与AI协作的方法论。

---

## 🎬 最后的建议：养成PDCA习惯

如果你今年要做很多AI+编程的事儿，建议这样做：

1. **项目开始** - 写Plan版本，记录初始设想
2. **项目完成** - 沉淀新的Plan版本，对比改进
3. **持续积累** - 每个项目的设计模式、提示词模板、代码片段都保留
4. **形成风格** - 重复10-15个项目后，你的"AI应用风格"就成型了

这比单纯地"做完一个项目"高效100倍。
